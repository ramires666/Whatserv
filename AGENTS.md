# Правила работы над проектом

- Для независимых подзадач использовать максимум доступных быстрых и экономичных агентов, если это разрешено инструкциями текущего сеанса.
- Главный агент всегда остаётся архитектором и оркестратором: определяет контракты, раздаёт непересекающиеся области файлов, интегрирует изменения и отвечает за итоговое качество.
- Результат каждого агента подлежит ревью главным агентом. Работа не считается завершённой без подходящих тестов, проверки безопасности и общей интеграционной проверки.
- Не позволять агентам одновременно редактировать одни и те же файлы. Общие контракты и инфраструктурные файлы меняет главный агент.
- При выборе между скоростью и корректностью итогового кода приоритет имеют корректность, безопасность данных и воспроизводимость.
- Для архитектурных схем, диаграмм последовательностей, состояний и зависимостей в Markdown использовать Mermaid. Текстовые/ASCII-диаграммы допустимы только если Mermaid недоступен или пользователь явно запросил другой формат.
